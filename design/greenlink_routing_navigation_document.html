# GreenLink Routing & Navigation Architecture ## 1. Overview GreenLink utilizes a **Workspace-Based Routing** model. Instead of a single, deeply nested hierarchy, the application is divided into top-level functional "Workspaces" (e.g., Dashboard, Golf Ops, Finance). Each workspace acts as a focused environment with its own internal navigation and state. ## 2. Path Structure (Proposed) The application follows a standard RESTful path naming convention for clear mapping between the UI and the underlying data. ### Staff/Admin Workspaces (`/admin/...`) * **Overview:** `/admin/dashboard` * **Golf Operations:** * Tee Sheet (Daily): `/admin/golf/tee-sheet` * Booking Rules: `/admin/golf/settings` * **Members / People:** * Directory: `/admin/members` * Individual Profile: `/admin/members/:id` * **Finance:** * Cashbook: `/admin/finance/cashbook` * Export History: `/admin/finance/exports` * **Communications:** * Workspace: `/admin/communications` * Message Composer: `/admin/communications/new` * **Pro Shop (POS):** * Terminal View: `/admin/pos` * Transaction History: `/admin/pos/transactions` ### Player App (`/player/...`) * **Home:** `/player/home` * **Booking:** `/player/book` * **News/Club:** `/player/news` * **Messages:** `/player/messages` * **Profile:** `/player/profile` ## 3. Navigation Component Logic The navigation is driven by two primary components that handle the routing triggers. ### SideNavBar (Staff Desktop) - **State Management:** Tracks the `activePath`. - **Interaction:** Clicking a tab triggers a route change (e.g., `router.push('/admin/finance')`). - **Visual Feedback:** The active tab receives a high-contrast border and background shift (using our `Evergreen Operational` theme tokens). ### BottomNavBar (Player Mobile-First) - **State Management:** Tracks active index (0-4). - **Interaction:** Immediate context switch between core player functions. ## 4. Implementation Example (Conceptual) In the HTML code provided for each screen, navigation is handled via `` tags with explicit paths. For a production SPA (Single Page Application), these would be replaced by framework-specific router links: ```html Finance ``` ## 5. Flow Logic 1. **Direct Entry:** Users hitting `/admin` are redirected to `/admin/dashboard`. 2. **Cross-Workspace Linking:** Critical alerts (e.g., "Finance Export Needed") on the Dashboard link directly to the relevant sub-path (e.g., `/admin/finance/cashbook`). 3. **POS Mode:** The Pro Shop POS is treated as a "Focused Mode" that hides the standard navigation to maximize screen real estate for transactions.
